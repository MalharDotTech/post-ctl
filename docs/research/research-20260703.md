# socialctl Research — 2026-07-03

Condensed findings from three parallel research tracks (Postiz internals, Buffer API, platform API/auth landscape) plus a browser-automation feasibility check. This is the evidence base behind `design.md`. Facts below were verified against primary sources on 2026-07-03; platform terms churn — re-verify pricing/limits at build time.

---

## 1. Postiz (gitroomhq/postiz-app) — what self-hosting actually buys

**Architecture:** NestJS + PostgreSQL + Redis + **Temporal** (workflow engine). Providers in `libraries/nestjs-libraries/src/integrations/social/` — 34 files, ~15.3k LOC; with shared auth plumbing and Temporal workflows the auth+provider layer is **~17k LOC across ~45 files** (~450 LOC average per provider; TikTok 1,115, Instagram 1,096, LinkedIn 982+919, Facebook 887).

**Provider interface:** `SocialProvider` = `IAuthenticator` (`generateAuthUrl`, `authenticate`, `refreshToken`) + posting + optional analytics/comments, plus behavior flags (`isBetweenSteps`, `oneTimeToken`, `refreshCron`, `isChromeExtension`, `customFields`). Shared `SocialAbstract.fetch()` retries 429/5xx and converts 401 → a Temporal `refresh_token` failure that drives the refresh machinery.

**Token lifecycle:** tokens stored **plaintext** in Postgres. Reactive refresh on 401 during posting; proactive refresh via per-integration Temporal workflows sleeping until `tokenExpiration` (only for Meta-style `refreshCron` providers). On refresh failure: `refreshNeeded=true`, email the user, disconnect the channel.

**Key gotchas found in source:**
- **Facebook & Instagram(-via-Facebook) providers have STUB `refreshToken()`** (`return {accessToken:''}`) — they rely on the 60-day long-lived token minted at connect time; when it lapses the user must manually reconnect. This is the "auth again to post" behavior observed when evaluating Postiz — an implementation artifact, not a platform requirement.
- X provider uses **OAuth 1.0a** (for v1.1 media upload), custom request signing.
- Instagram provider stores two tokens concatenated in one DB field (`page___user`, split on `___`).
- Genuinely unofficial providers: Skool (Chrome-extension cookie harvesting), MeWe (undocumented dev-preview endpoint). Everything else is official APIs.

**Self-hosting burden:** every OAuth provider requires the self-hoster's **own developer app** + env vars + container restart. Docs require Meta Business app with 6–7 advanced permissions and Live mode for public use; LinkedIn token *refresh* gated on Advertising API partner form; TikTok needs a domain-verified public HTTPS site because it pulls media by URL.

**Issue-tracker pain (recurring):** exact-match redirect-URI failures (#891, #1122), Meta scope drift (#1216), IG "media could not be fetched" behind reverse proxies (#1472), GMB daily manual refresh from Google testing-mode limits (#1091), and the worst case — a posting loop that got a user's Instagram account **permanently disabled** (#832).

**Conclusion:** Postiz ≠ auth escape hatch. Same dev-app ceremony as building ourselves, plus a 4-service stack and 17k LOC of moving provider code, most implementing features (analytics/comments/scheduling UI) out of our scope.

## 2. Buffer API — 2019 shutdown, 2026 beta

- **Old public REST API:** deprecated 2019 (had 47k clients; Cambridge-Analytica-era compliance + strategy). **Closed to new developer registrations** — no client_id obtainable. Most "Buffer API" references online describe this dead surface.
- **New GraphQL API** (`api.buffer.com`): **public beta**, auth via **personal API key** (Bearer header, minted in account settings — no OAuth, no dev app). Operations: `createPost` (immediate or `customScheduled`+`dueAt`), delete, list channels/org/ideas. **11 channels**: X, Instagram, Facebook, LinkedIn, TikTok, Threads, YouTube, Pinterest, Google Business Profile, Mastodon, Bluesky. Buffer holds all the audited platform apps — user connects channels in Buffer's UI.
- **Limits:** images via `imageUrl` (public URL only); **native video upload not yet exposed** ⇒ YouTube/TikTok effectively blocked through it. Plan-gated: Free = 1 API key, ~100 req/24h, 3 channels, 10 queued posts/channel, lifetime cap 8 channel connections; paid ~$5–6/channel/mo. Third-party OAuth (onboarding other users) not enabled.
- **Risk:** beta, no SLA, and Buffer has killed an API before.
- **Verdict:** one optional `token`-kind provider (~150 LOC) — cheapest zero-dev-app fallback for X/IG/FB/LinkedIn text+image posting; never the foundation.

## 3. Big-5 platform auth & posting APIs (mid-2026 state)

| Platform | Auth flow | Dev-app setup (once) | Token lifecycle | Re-login cadence | Cost | Hard constraints |
|---|---|---|---|---|---|---|
| **X** | OAuth2 auth-code + **PKCE** (user context) | Dev account + app; **card required** — free tier discontinued Feb 2026, pay-per-use default | ~2h access, rotating refresh token | **Never** (silent refresh) | **$0.015/post; $0.20/post containing a link**; $0.005/read | Media upload path historically OAuth 1.0a (v1.1); verify v2 media endpoints at build time |
| **LinkedIn** | OAuth2 auth-code, **client secret, no PKCE**; scope `w_member_social` (self-serve "Share on LinkedIn" + "Sign In with OpenID Connect" products) | Self-serve app, no review for own-profile posting | 60-day access token; **refresh tokens only for Marketing Developer Platform partners** | **Every 60 days** (manual re-OAuth; unfixable by tooling) | Free | Member profile only; org pages need Community Management review (~2–4 wks) |
| **Facebook** | Meta OAuth via own app in **Development mode** (no App Review when only role-holding accounts use it) | Meta app + add self as admin | Long-lived user token 60d → **Page tokens effectively non-expiring** | **Effectively never** | Free | **Pages only** — personal-timeline posting removed from API in 2018, policy-level |
| **Instagram** | Meta OAuth — "Instagram API with Instagram Login" (no FB Page link needed) or classic Graph-via-Page; same dev-mode loophole | Same Meta app; **account must be Business/Creator** | Long-lived 60d, refreshable via `ig_refresh_token` any time after 24h age | **Never IF used ≥ once/60 days**; idle beyond ⇒ re-login | Free | Media **required** (no text-only); publish **pulls media from public HTTPS URL** ⇒ staging needed; ~50 posts/24h API cap |
| **YouTube** | Google OAuth2 installed-app + PKCE; consent screen **must be "Production"** (Testing mode kills refresh tokens after 7 days) | GCP project + consent screen + enable Data API v3 | Durable refresh token once in Production | **Never** | Free | **Unaudited API projects: uploads forced Private, no appeal** (projects created after Jul 2020); compliance-audit questionnaire is the unlock. Quota 10k units/day; upload=1,600 ⇒ ~6 uploads/day |

Cross-cutting:
- **Meta dev-mode loophole** is the load-bearing fact for personal tools: an app that only serves accounts holding a role on it never needs App Review or business verification.
- **Refresh collapses to 4 strategies** (none/standard/exchange/reauth) — see design.md; only LinkedIn requires recurring human action.
- Instagram + Buffer share the **pull-media-from-URL** model ⇒ one staging solution serves both.

## 4. Browser-session automation — rejected as core path

Evaluated: Playwright persistent profiles, CDP-attach to real Chrome, cookie extraction, and private-API libs (twikit/instagrapi-class).

- **Instagram & LinkedIn actively restrict accounts**, triggered by device-fingerprint/session mismatch more than volume (IG `challenge_required` after 2–3 logins with mismatched fingerprints; LinkedIn ML detection, real account restrictions). Cookie-injection into fresh contexts is the highest-risk pattern.
- X tolerates light automation but has an official API, making the risk pointless.
- Selector churn on all five platforms = unbounded maintenance; LLM-driven agents (browser-use/Stagehand) trade selector rot for cost/nondeterminism/behavioral detectability, not zero maintenance.
- If ever added: opt-in provider, **CDP-attach to the user's real logged-in Chrome only**, accessibility-tree targeting, never default, never Instagram/LinkedIn.

## 5. Aggregator landscape (fallback options)

- **Ayrshare** — incumbent, $149/mo for 1 profile. Out of budget class.
- **Late (getlate.dev)** — dev-focused, free tier (2 profiles/~10 posts/mo), ~$13–19/mo paid, ~13 platforms; holds the TikTok/YouTube audits.
- Crowded 2025–26 field of flat-priced alternatives (Post for Me, bundle.social, Zernio, PostFast — the latter ships an MCP server).
- Architectural stance: aggregators are ordinary `token`-kind providers behind the same interface — adopt/drop without redesign.
